- 保存最小响应样例，覆盖 `queryLeftNewDTO` 字段解析和高铁/普速票价字段映射。
- 选择一个数据源实现适配器。
- 增加数据源状态接口。
- 增加缓存和失败降级。

单元测试：

- 站名到电报码映射解析测试。
- `queryLeftNewDTO` 字段解析测试。
- `swz_price`、`zy_price`、`ze_price`、`yz_price`、`yw_price`、`rw_price`、`gr_price` 到 `SeatPrice` 映射测试。

集成测试：

- 数据源正常时返回真实格式归一化后的车次段。

验收标准：

- 系统可以在 Mock 数据源和真实数据源之间切换。
- 真实接口不可用时，Mock 数据源和核心搜索算法不受影响。

## 阶段 7：真实数据源中转能力设计与实现（已落地）

- 建立站点元数据结构，至少包含站名、电报码、经纬度和枢纽分。
- 实现候选中转站生成器：空间带状剪枝、朝向剪枝、枢纽优先排序。
- 为真实 provider 增加单次搜索请求预算，避免一次查询无约束放大外部请求。
- 将 OD 查询缓存从进程内 TTL 扩展为可选 SQLite 持久化缓存。
- 保持缓存 key 与 12306 OD 查询粒度一致：`provider:date:from_station:to_station`。
- 修改 `Railway12306PublicPriceProvider.candidate_transfer_stations`，从本地候选生成器返回受控候选站，而不是返回空列表。
- 保持 `RouteSearchService` 不直接依赖 12306 URL、字段或请求头。

单元测试：

- 候选站空间剪枝测试：偏离 AB 线段过远的站点被剔除。
- 候选站朝向剪枝测试：投影在线段外的站点被剔除。
- 枢纽优先排序测试：同等空间条件下高枢纽分站点优先。
- 请求预算测试：预算耗尽后不再触发远程查询。
- SQLite OD 缓存测试：命中、过期、损坏数据降级。

集成测试：

- 使用 fake 真实 provider 验证一次中转搜索不会超过请求预算。
- 缓存命中时重复查询不再请求远程 client。
- 无中转结果时仍返回可用直达方案。

验收标准：

- 真实 provider 可以返回受控的一次中转候选方案。
- 单次搜索远程 OD 请求数可配置、可测试、可观测。
- 缓存可降低重复查询请求量。
- 真实数据源失败时 Mock provider 和核心搜索算法不受影响。
