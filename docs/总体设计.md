# 总体设计

## 技术栈

- 前端：React、TypeScript、Vite、Tailwind CSS、shadcn/ui。
- 后端：FastAPI、Pydantic、httpx、pytest。
- 数据源：Mock 数据源、GitHub 开源项目适配器、可选 MCP 查询适配器。
- 缓存：第一阶段可用内存缓存；后续可替换为 Redis。

## 后端模块

### domain

定义稳定领域模型：

- `Station`：车站。
- `TrainSegment`：单段车次。
- `TicketPrice`：席别与价格。
- `RouteQuery`：查询条件。
- `TransferPlan`：中转方案。

### adapters

定义统一数据源接口：

```python
class TrainDataProvider:
    async def search_segments(self, query: SegmentQuery) -> list[TrainSegment]:
        ...
```

具体适配器：

- `MockTrainDataProvider`：本地样例数据，用于开发和测试。
- `GithubTrainDataProvider`：封装开源项目或公开接口。
- `McpTrainDataProvider`：封装 MCP 工具查询结果。

### services

- `RouteSearchService`：组合直达和中转方案。
- `PricingService`：归一化不同席别和票价。
- `ProviderHealthService`：数据源状态检查。
- `CacheService`：缓存查询结果。

## API 设计

### 健康检查

`GET /api/health`

### 站点搜索

`GET /api/stations/search?q=北京`

### 方案查询

`POST /api/routes/search`

请求示例：

```json
{
  "from_station": "北京",
  "to_station": "上海",
  "date": "2026-07-01",
  "max_transfers": 1,
  "min_transfer_minutes": 30
}
```

返回示例：

```json
{
  "query_id": "20260701-bj-sh",
  "source": "mock",
  "updated_at": "2026-06-25T09:00:00+08:00",
  "plans": [
    {
      "total_price": 388.5,
      "total_duration_minutes": 520,
      "transfer_stations": ["南京南"],
      "segments": []
    }
  ]
}
```

## 前端页面

- 搜索栏：出发地、目的地、日期、最短换乘时间。
- 结果列表：总价、总耗时、出发到达时间、换乘站、车次段。
- 结果排序：固定按最低总价排序。
- 状态显示：加载中、无结果、数据源错误、数据更新时间。

## 算法概要

第一阶段支持 0 到 1 次中转：

1. 查询直达车次。
2. 获取候选中转站。
3. 查询 `出发站 -> 中转站` 和 `中转站 -> 目的站` 两段车次。
4. 过滤不满足换乘时间、时间顺序、总耗时约束的组合。
5. 计算每个方案总价、总耗时、换乘等待时间。
6. 按最低总价输出前 N 个方案，总耗时和换乘次数仅作为同价辅助排序。

后续可扩展为时间相关图搜索和 Pareto 最优剪枝。
