# 时间扩展图多重搜索架构原子化实施计划

## 0. 执行原则

本文档用于交给能力较低的模型逐步执行。必须按切片顺序推进，不要跨切片混改。

### 总目标

在不删除、不破坏现有候选中转站 OD 搜索的前提下，引入可由环境变量切换的多重搜索架构：

```text
ROUTE_SEARCH_ENGINE=candidate       -> 旧搜索
ROUTE_SEARCH_ENGINE=time-expanded   -> 新本地时间扩展图搜索
```

### 硬约束

1. 默认行为必须保持旧搜索。
2. 新时间扩展图搜索在线阶段不得调用任何远程接口。
3. 新时间扩展图搜索不得调用 `TrainDataProvider.search_segments()`。
4. 不删除 `CandidateTransferStationGenerator`。
5. 不删除 `static-with-remote-fallback`，但它只属于旧搜索链路。
6. 不把同车长 OD 票价拆成相邻区间价格相加。
7. 每个切片完成后必须运行对应测试。

### 推荐验证命令

后端全量测试：

```powershell
cd backend
python -m pytest -q
```

针对性测试示例：

```powershell
cd backend
python -m pytest tests/test_route_search_engine_factory.py -q
python -m pytest tests/test_time_expanded_route_search.py -q
```

```python
import os

from app.adapters.base import TrainDataProvider
from app.services.route_search import RouteSearchService
from app.services.route_search_engine import RouteSearchEngine


def create_route_search_engine(provider: TrainDataProvider) -> RouteSearchEngine:
    name = os.getenv("ROUTE_SEARCH_ENGINE", "candidate").strip().lower()
    if name in {"", "candidate", "od-candidate", "legacy"}:
        return RouteSearchService(provider)
    if name in {"time-expanded", "time_expanded"}:
        raise ValueError("ROUTE_SEARCH_ENGINE=time-expanded is not implemented yet")
    raise ValueError(f"Unsupported ROUTE_SEARCH_ENGINE: {name}")
```

## 切片 7：实现时间扩展图搜索引擎最小版本

### 目标

新增 `TimeExpandedRouteSearchEngine`，从本地 `train_od_price_snapshot` 构建边并执行 Dijkstra。

### 修改文件

新增：

```text
backend/app/services/time_expanded_route_search.py
backend/tests/test_time_expanded_route_search.py
```

### 实现方向

第一版不必构建完整事件节点类，只需把每条 `train_od_price_snapshot` 当成一条乘车边：

```text
from_station at depart_at -> to_station at arrive_at, weight=price
```

状态包含：

```python
station
arrive_at
total_price
segments
```

扩展规则：

1. 初始状态在 `query.from_station`，`arrive_at=None`。
2. 只选择 `edge.from_station == state.station` 的边。
3. 第一段要求 `edge.depart_at.date()` 在查询日期窗口内，且通常应为 `query.date` 或后续由窗口控制。
4. 非第一段要求 `edge.depart_at >= state.arrive_at + min_transfer_minutes`。
5. `len(segments) - 1 <= max_transfers`。
6. 到达目的地后构造 `TransferPlan`。

### 伪代码

```python
class TimeExpandedRouteSearchEngine:
    def __init__(self, repository, provider="12306-public-price"):
        self.repository = repository
        self.provider = provider

    @property
    def source(self):
        return "time-expanded"

    @property
    def status(self):
        return {
            "provider": self.provider,
            "search_engine": "time-expanded",
            "status": "ok",
            "online_remote_io": False,
        }

    async def search(self, query):
        started_at = perf_counter()
        plans = await self._search_plans(query)
        return RouteSearchResponse(...)

    async def _search_plans(self, query):
        service_dates = [query.date, query.date + timedelta(days=1), query.date + timedelta(days=2)]
        rows = self.repository.query_train_od_prices(self.provider, service_dates)
        edges_by_station = group_by_from_station(rows)
        queue = [(Decimal("0"), 0, initial_state)]
        while queue:
            price, _, state = heapq.heappop(queue)
            for edge in edges_by_station.get(state.station, []):
                if not can_take(edge, state, query):
                    continue
                next_segments = (*state.segments, build_segment(edge))
                if edge.to_station == query.to_station:
                    plans.append(build_plan(next_segments))
                else:
                    push next state
        return sorted(plans, key=plan_sort_key)[:10]
```

### 流式接口伪代码

第一版允许流式接口一次性返回最终 snapshot：

```python
async def stream_snapshots(self, query):
```
