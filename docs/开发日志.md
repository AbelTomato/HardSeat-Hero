# 开发日志

## 2026-06-25 初始化构建与范围收敛

### 当前目标

项目目标已收敛为：根据出发地、目的地和出行日期，查询最低价火车直达或一次中转方案列表。

系统不提供“最短耗时”或“最少中转”排序能力。总耗时、换乘站和换乘等待时间只作为方案辅助信息展示。

### 当前重要约束

- 项目只查询最低价中转方案，不做最短耗时或最少中转排序。
- 官方实时数据暂不作为第一阶段强依赖。
- GitHub 开源项目或 MCP 查询能力后续必须通过 `TrainDataProvider` 适配，不应直接耦合到搜索算法。
- 当前数据为 Mock 数据，只用于算法和 UI 闭环验证。

### 下一步建议

1. 增加数据源缓存层，为真实数据源接入做准备。
2. 调研 GitHub/MCP 可用数据源，记录许可证、维护状态、调用方式和失效风险。
3. 实现第一个真实数据源适配器，并保持 Mock 数据源可切换。
4. 增加前端接口错误细分展示，例如后端不可用、数据源不可用、无方案。
