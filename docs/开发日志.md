# 开发日志

## 2026-06-25 初始化构建与范围收敛

### 当前目标

项目目标已收敛为：根据出发地、目的地和出行日期，查询最低价火车直达或一次中转方案列表。

系统不提供“最短耗时”或“最少中转”排序能力。总耗时、换乘站和换乘等待时间只作为方案辅助信息展示。

### 当前重要约束

- 项目只查询最低价中转方案，不做最短耗时或最少中转排序。
- 官方实时数据暂不作为第一阶段强依赖。
- GitHub 开源项目或 MCP 查询能力后续必须通过 `TrainDataProvider` 适配，不应直接耦合到搜索算法。
- 当前数据为 Mock 数据，只用于算法和 UI 闭环验证。

### 下一步建议

1. 增加数据源缓存层，为真实数据源接入做准备。
2. 调研 GitHub/MCP 可用数据源，记录许可证、维护状态、调用方式和失效风险。
3. 实现第一个真实数据源适配器，并保持 Mock 数据源可切换。
4. 增加前端接口错误细分展示，例如后端不可用、数据源不可用、无方案。

- 后续接入真实数据源后，可替换或扩展为 Redis，但不应改变 `RouteSearchService` 对 Provider 的调用边界。

## 2026-06-25 票价获取方案调研

### 已完成成果

- 确认 `Parse12306` 可提供车站、车次和经停时刻表抓取思路，但不提供票价。
- 确认 `China-rail-way-stations-data` 只解析车站和车次始发终到数据，且 README 标明数据过期，不提供票价。
- 评估用户提供的 `leftTicket/queryZ` 与 `queryTicketPrice` 示例代码：链路可行，但代码存在交互式输入、全局状态、席别分支错误和错误处理不足，不能直接接入服务端。
- 新增 `docs/票价获取方案.md`，记录 12306 余票和票价接口链路、字段映射、缓存策略和失败降级。
- 更新 `docs/数据源调研.md` 和 `docs/实施计划.md`，把票价接口探测纳入真实数据源接入阶段。
- 新增 `backend/scripts/probe_12306_pricing.py`，用于手动探测站点电报码、余票和票价接口。
- 后续验证发现旧 `leftTicket/queryZ`、`leftTicket/query`、`leftTicket/queryA` 当前返回 HTML，不适合作为主路径。
- 已确认 12306 官方公布票价页面 `queryPublicIndex.html` 使用 `leftTicketPrice/queryAllPublicPrice`，该接口可直接返回车次、时刻和公布票价 JSON。
- 探测脚本已改为使用 `leftTicketPrice/queryAllPublicPrice`，并支持高铁动车与 K/Z 普速车价格字段解析。

### 当前约束

- 12306 相关接口不是公开稳定 API，字段和 header 要求可能变化。
- 探测脚本不接入主 API，不影响 Mock 数据源和核心搜索算法。
- 正式适配器仍需补充 fixture、单元测试、缓存、`infoAll_list` 扩展解析和配置化 provider 切换。
