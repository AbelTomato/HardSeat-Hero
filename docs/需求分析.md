# 需求分析

## 目标

实现一个根据出发地、目的地和出行日期，给出低价火车中转方案列表的 Web 应用。

系统第一阶段不以官方实时票务数据为强依赖，而是先建立可插拔数据源、查询算法、后端 API 和前端展示闭环。真实数据源接入作为独立里程碑推进。

## 用户场景

- 用户输入出发地、目的地、出行日期。
- 系统返回直达和中转方案。
- 用户查看按最低总价排序的中转方案。
- 用户查看每个方案的车次段、出发到达时间、换乘站、等待时间、总价和数据更新时间。

## MVP 范围

### 必做

- 站点输入和日期输入。
- 0 次或 1 次中转方案搜索。
- 最低总价排序。
- 最短换乘时间过滤。
- Mock 数据源。
- FastAPI 查询接口。
- React + shadcn/ui 查询与结果展示页面。
- 后端单元测试覆盖核心算法。

### 暂缓

- 用户登录。
- 收藏方案。
- 支付或购票跳转。
- 自动抢票。
- 多日价格趋势。
- 2 次及以上中转。
- 全量车站图谱优化搜索。

## 非功能需求

- 数据源必须可替换，核心算法不依赖具体第三方项目返回格式。
- 真实数据源失败时，接口应返回明确错误或降级结果。
- 查询结果必须包含数据来源和更新时间。
- 后端接口使用结构化 schema，便于前后端联调和测试。
- 算法逻辑应可独立单元测试。

## 风险

- 官方票务网站存在反爬、验证码、登录态和使用条款限制。
- GitHub 开源项目可能失效、接口不稳定或许可证不适合继续使用。
- 近实时票价和余票变化快，结果只能作为参考。
- 当前产品目标限定为最低价方案，耗时和换乘信息只作为辅助展示。
